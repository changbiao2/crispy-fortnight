# 陈阳 - 任务清单

## 基本信息
- **角色**：详细设计师
- **负责模块**：详细设计、代码生成、文档整合

## 一、详细设计任务

### 1.1 总体结构设计
- 绘制系统总体结构设计图
- 确定系统分层架构（表示层、业务逻辑层、数据访问层）

### 1.2 泳道图设计（使用EA绘制）
| 序号 | 泳道图名称 | 说明 |
|------|-----------|------|
| 1 | 用户购物流程泳道图 | 包含用户、商品、购物车、订单等多个泳道 |
| 2 | 订单处理流程泳道图 | 包含用户、系统、管理员、物流等泳道 |
| 3 | 库存管理流程泳道图 | 包含管理员、系统、数据库等泳道 |

### 1.3 状态图设计（使用EA绘制）
| 序号 | 状态图名称 | 状态列表 |
|------|-----------|----------|
| 1 | 订单状态图 | 待支付 → 已支付 → 待发货 → 已发货 → 已完成 / 已取消 |
| 2 | 商品状态图 | 草稿 → 已上架 → 已下架 |
| 3 | 用户状态图 | 未激活 → 正常 → 禁用 |

## 二、类图整合与设计模式

### 2.1 整合所有模块类图
- 收集李昶标、邓耀华、张俊杰的类图
- 统一类图风格和命名规范
- 确保类之间的关系正确

### 2.2 设计模式应用
| 设计模式 | 应用场景 | 说明 |
|----------|----------|------|
| 工厂模式 | 对象创建 | 商品、订单、用户等对象的创建 |
| 外观模式 | 接口简化 | 为复杂的子系统提供统一入口 |
| 单例模式 | 配置管理 | 数据库连接池、配置管理器 |
| DAO模式 | 数据访问 | 数据访问层统一接口 |

### 2.3 工厂模式与外观模式结合类图
- 绘制工厂模式类图
- 绘制外观模式类图
- 绘制两者结合的综合类图

## 三、代码生成（使用EA）

### 3.1 生成步骤
1. 在EA中完成所有类图设计
2. 选择要生成代码的类
3. 设置代码生成选项（语言：Java）
4. 生成项目初始代码框架

### 3.2 生成的代码结构
```
src/
├── main/
│   ├── java/
│   │   ├── com/shopping/
│   │   │   ├── entity/          # 实体类
│   │   │   ├── dao/             # 数据访问层
│   │   │   ├── service/         # 业务逻辑层
│   │   │   ├── controller/      # 控制器层
│   │   │   └── util/            # 工具类
│   │   └── ...
│   └── resources/
│       ├── mapper/              # MyBatis映射文件
│       └── application.yml      # 配置文件
└── ...
```
