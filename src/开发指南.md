# 网上购物网站 - 完整开发指南

## 一、项目概述

### 1.1 项目简介

本项目是一个完整的 B2C 网上购物商城系统，包含前台商城和后台管理系统。采用前后端分离架构，支持用户注册登录、商品浏览、购物车管理、订单处理、在线客服等核心电商功能。

**适用场景：**
- 课程设计/毕业设计
- 电商系统学习
- 全栈开发实践
- 团队协作项目

### 1.2 技术栈选型

#### 前端技术栈
| 技术 | 版本 | 说明 |
|------|------|------|
| Vue.js | 3.x | 渐进式 JavaScript 框架（CDN 引入） |
| Tailwind CSS | 3.x | 原子化 CSS 框架（CDN 引入） |
| Font Awesome | 6.x | 图标库（CDN 引入） |
| Chart.js | 4.x | 数据可视化图表库 |
| Axios | 1.x | HTTP 客户端（可选） |

#### 后端技术栈
| 技术 | 版本 | 说明 |
|------|------|------|
| Node.js | 18+ | JavaScript 运行时环境 |
| Express | 4.x | Web 应用框架 |
| CORS | 2.x | 跨域资源共享中间件 |
| Body-parser | 1.x | JSON 解析中间件 |

#### 数据存储
- **当前**: 内存模拟（重启后数据重置）
- **可扩展**: MySQL / MongoDB / Redis

**关键代码示例:**
```javascript
// 登录功能
async function doLogin() {
  const res = await fetch(`${API_BASE}/api/auth/login`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ username, password })
  });
  const data = await res.json();
  
  if (data.code === 200) {
    localStorage.setItem('token', data.data.token);
    localStorage.setItem('user', JSON.stringify(data.data.user));
  }
}
```

## 五、数据库设计

### 5.1 数据模型总览

```
User（用户）
  ├─ id: 主键
  ├─ username: 用户名（唯一）
  ├─ password: 密码
  ├─ nickname: 昵称
  ├─ email: 邮箱
  ├─ phone: 手机号
  ├─ avatar: 头像 URL
  ├─ role: 角色（user/admin）
  ├─ status: 状态（active/disabled）
  └─ createTime: 创建时间

Product（商品）
  ├─ id: 主键
  ├─ name: 商品名称
  ├─ price: 现价
  ├─ originalPrice: 原价
  ├─ image: 主图 URL
  ├─ category: 分类
  ├─ description: 描述
  ├─ stock: 库存
  ├─ sales: 销量
  ├─ status: 状态（on/off）
  └─ createTime: 上架时间

Order（订单）
  ├─ id: 主键
  ├─ orderNo: 订单编号（唯一）
  ├─ userId: 用户 ID（外键）
  ├─ items: 商品列表 [{productId, name, price, quantity, image}]
  ├─ totalAmount: 总金额
  ├─ status: 订单状态
  ├─ address: 收货地址 {name, phone, address}
  ├─ remark: 备注
  ├─ createTime: 创建时间
  ├─ payTime: 支付时间
  ├─ shipTime: 发货时间
  └─ completeTime: 完成时间

Message（留言）
  ├─ id: 主键
  ├─ userId: 用户 ID
  ├─ username: 用户名
  ├─ content: 内容
  ├─ replies: 回复列表 [{userId, username, content, createTime}]
  └─ createTime: 创建时间
```

## 六、API 接口设计规范

### 6.1 RESTful 风格

**基本原则:**
- 资源使用名词，避免动词
- 使用 HTTP 方法表示操作
- 返回统一格式

**HTTP 方法:**
| 方法 | 操作 | 示例 |
|------|------|------|
| GET | 查询 | GET /api/products |
| POST | 新增 | POST /api/orders |
| PUT | 修改 | PUT /api/user/profile |
| DELETE | 删除 | DELETE /api/cart/:id |

---

### 6.2 响应格式规范

**成功响应:**
```javascript
{
  "code": 200,
  "message": "操作成功",
  "data": { ... }
}
```

**列表响应:**
```javascript
{
  "code": 200,
  "message": "获取成功",
  "data": {
    "list": [...],
    "total": 100,
    "page": 1,
    "limit": 10,
    "totalPages": 10
  }
}
```

**错误响应:**
```javascript
{
  "code": 400,
  "message": "参数错误"
}
```

**状态码说明:**
| 状态码 | 含义 |
|--------|------|
| 200 | 成功 |
| 400 | 参数错误 |
| 401 | 未授权（需登录） |
| 403 | 禁止访问（权限不足） |
| 404 | 资源不存在 |
| 500 | 服务器内部错误 |

---

### 6.3 Token 认证机制

**Token 生成:**
```javascript
// 登录成功后生成
const token = 'token_' + Math.random().toString(36).substr(2) + Date.now();
data.tokens[token] = userId;
```

**Token 使用:**
```javascript
// 前端请求头
headers: {
  'Authorization': 'Bearer ' + token
}
```

**Token 验证中间件:**
```javascript
// backend/middleware/auth.js
const verifyToken = (req, res, next) => {
  const token = req.headers.authorization?.replace('Bearer ', '');
  const userId = data.tokens[token];
  
  if (!userId) {
    return res.status(401).json({ code: 401, message: '请先登录' });
  }
  
  req.userId = userId;
  next();
};
```

## 八、常见问题解答（FAQ）

### Q1: 如何修改端口号？
**A:** 编辑 `backend/config/database.js`，修改 `port: 3000` 为你想要的端口。

### Q2: 数据重启后丢失怎么办？
**A:** 当前使用内存模拟，重启会重置。如需持久化，可改为：
- SQLite（轻量级）
- MySQL（生产级）
- MongoDB（文档型）

### Q3: 如何实现文件上传？
**A:** 使用 `multer` 中间件：
```javascript
const multer = require('multer');
const upload = multer({ dest: 'uploads/' });
app.post('/upload', upload.single('file'), (req, res) => {
  // req.file.path 即为上传文件路径
});
```

### Q4: 跨域问题如何解决？
**A:** 已在 `server.js` 中配置 CORS 中间件：
```javascript
app.use(cors());
```

### Q5: 如何部署到服务器？
**A:** 
1. 购买云服务器（阿里云/腾讯云）
2. 安装 Node.js
3. 上传代码
4. 使用 PM2 管理进程：
```bash
npm install -g pm2
pm2 start server.js --name shop
```
