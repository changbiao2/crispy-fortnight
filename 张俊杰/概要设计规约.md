## 1. 引言

### 1.1 编写目的

本文档是网上购物网站项目的概要设计规约，旨在确定系统的整体架构、模块划分、技术选型等宏观设计方案，为详细设计和开发实施提供指导。

### 1.2 项目背景

随着电子商务的蓬勃发展，网上购物已成为主流消费方式。本项目旨在构建一个功能完善、性能优良、安全可靠的B2C网上购物平台，满足用户在线购物和商家管理的需求。

### 1.3 读者对象

- 项目开发人员
- 详细设计人员
- 测试人员
- 项目管理人员

### 1.4 术语定义

| 术语 | 说明 |
|------|------|
| B2C | Business-to-Consumer，企业对消费者的电子商务模式 |
| MVC | Model-View-Controller，模型-视图-控制器架构模式 |
| RESTful | 基于HTTP协议的API设计风格 |
| JWT | JSON Web Token，JSON网络令牌 |
| ORM | Object-Relational Mapping，对象关系映射 |

## 2. 系统概述

### 2.1 系统目标

构建一个具有以下特点的网上购物网站：
- **功能完善**：覆盖商品展示、购物车、订单管理、支付等核心功能
- **性能优良**：支持高并发访问，响应速度快
- **安全可靠**：数据安全、交易安全、用户隐私保护
- **易用性强**：界面友好，操作简便

### 2.2 系统范围

#### 2.2.1 包含功能
- 前台系统：用户注册登录、商品浏览搜索、购物车、订单管理
- 后台系统：商品管理、订单管理、用户管理、库存管理、数据统计
- 辅助系统：留言系统、在线客服

#### 2.2.2 系统边界
- 不包含第三方支付接口集成（采用模拟支付）
- 不包含物流系统对接（采用模拟物流状态）
- 不包含移动端APP（仅Web响应式）

### 2.3 设计约束

- 开发周期：8周
- 团队规模：4人
- 技术栈：Java + Spring Boot + MySQL

## 3. 系统架构设计

### 3.1 总体架构

系统采用前后端分离的分层架构：

```
┌─────────────────────────────────────────────────────────────┐
│                         前端层                               │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐      │
│  │   用户前端    │  │   管理后台    │  │   静态资源    │      │
│  │  (Vue/React) │  │  (Vue/React) │  │   (CDN)      │      │
│  └──────────────┘  └──────────────┘  └──────────────┘      │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼ HTTP/HTTPS
┌─────────────────────────────────────────────────────────────┐
│                         网关层                               │
│                    Nginx / API Gateway                       │
│              (负载均衡、反向代理、静态缓存)                    │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                        应用服务层                            │
│  ┌─────────────────────────────────────────────────────┐   │
│  │              Spring Boot 应用服务                    │   │
│  │  ┌─────────┐  ┌─────────┐  ┌─────────┐  ┌────────┐ │   │
│  │  │Controller│  │ Service │  │   DAO   │  │ Entity │ │   │
│  │  │  (控制层)│  │ (业务层)│  │(数据层) │  │(实体层)│ │   │
│  │  └─────────┘  └─────────┘  └─────────┘  └────────┘ │   │
│  └─────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                        数据存储层                            │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────────────┐  │
│  │   MySQL     │  │   Redis     │  │   文件存储/OSS      │  │
│  │  (主数据库)  │  │  (缓存会话)  │  │   (图片资源)       │  │
│  └─────────────┘  └─────────────┘  └─────────────────────┘  │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 架构分层说明

#### 3.2.1 前端层
- **用户前端**：面向消费者的购物界面
- **管理后台**：面向管理员的后台界面
- **技术栈**：Vue.js 3.x / React 18 + Element Plus / Ant Design

#### 3.2.2 网关层
- **Nginx**：反向代理、负载均衡、静态资源服务
- **功能**：请求路由、限流、SSL终端、缓存

#### 3.2.3 应用服务层
- **Controller层**：接收请求、参数校验、调用服务、返回响应
- **Service层**：业务逻辑处理、事务管理
- **DAO层**：数据访问、数据库操作
- **Entity层**：数据实体定义

#### 3.2.4 数据存储层
- **MySQL**：业务数据持久化存储
- **Redis**：会话缓存、热点数据缓存、分布式锁
- **OSS/文件存储**：商品图片、用户头像等静态资源

## 5. 数据库设计

### 5.1 设计原则

1. **第三范式（3NF）**：消除传递依赖，确保数据一致性
2. **统一命名**：表名使用小写下划线，字段名使用小写下划线
3. **主键策略**：采用自增ID或UUID
4. **索引优化**：为常用查询字段建立索引
5. **外键约束**：保持数据完整性

### 5.2 数据库架构

```
┌─────────────────────────────────────────────────────────────┐
│                      数据库架构                              │
├─────────────────────────────────────────────────────────────┤
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐         │
│  │   用户库    │  │   商品库    │  │   订单库    │         │
│  │  (user_db)  │  │(product_db) │  │ (order_db)  │         │
│  │             │  │             │  │             │         │
│  │ • user      │  │ • product   │  │ • order     │         │
│  │ • address   │  │ • category  │  │ • order_item│         │
│  │ • message   │  │ • cart      │  │             │         │
│  │ • chat_record│ │ • product_image│ │            │         │
│  └─────────────┘  └─────────────┘  └─────────────┘         │
│                                                              │
│  ┌─────────────┐  ┌─────────────┐                          │
│  │   库存库    │  │   管理库    │                          │
│  │(inventory_db)│  │  (admin_db) │                          │
│  │             │  │             │                          │
│  │ • inventory │  │ • admin     │                          │
│  │ • inventory_record│ • operation_log│                     │
│  │             │  │ • sales_statistics│                     │
│  └─────────────┘  └─────────────┘                          │
└─────────────────────────────────────────────────────────────┘
```

### 5.3 核心数据表

#### 5.3.1 用户相关表

| 表名 | 说明 | 核心字段 |
|------|------|----------|
| user | 用户表 | user_id, username, password, email, phone, status |
| address | 收货地址表 | address_id, user_id, receiver_name, phone, detail_address |
| message | 留言表 | message_id, user_id, content, parent_id, create_time |
| chat_record | 聊天记录表 | record_id, user_id, admin_id, content, sender_type |

#### 5.3.2 商品相关表

| 表名 | 说明 | 核心字段 |
|------|------|----------|
| product | 商品表 | product_id, name, category_id, price, stock, status |
| category | 分类表 | category_id, name, parent_id, sort_order |
| cart | 购物车表 | cart_id, user_id, product_id, quantity |

#### 5.3.3 订单相关表

| 表名 | 说明 | 核心字段 |
|------|------|----------|
| order | 订单表 | order_id, order_no, user_id, total_amount, status |
| order_item | 订单项表 | item_id, order_id, product_id, quantity, price |

#### 5.3.4 库存相关表

| 表名 | 说明 | 核心字段 |
|------|------|----------|
| inventory | 库存表 | inventory_id, product_id, quantity, warning_quantity |
| inventory_record | 库存记录表 | record_id, product_id, type, quantity, operator_id |

#### 5.3.5 管理相关表

| 表名 | 说明 | 核心字段 |
|------|------|----------|
| admin | 管理员表 | admin_id, username, password, role, status |
| operation_log | 操作日志表 | log_id, admin_id, operation, target, create_time |
| sales_statistics | 销售统计表 | stat_id, date, total_sales, order_count, new_user_count |

### 5.4 数据库范式分析

本数据库设计遵循第三范式（3NF）：
- **第一范式**：所有字段都是原子性的，不可再分
- **第二范式**：消除非主属性对主键的部分依赖
- **第三范式**：消除非主属性对主键的传递依赖

## 6. 接口设计

### 6.1 接口设计原则

1. **RESTful风格**：使用HTTP方法表示操作类型
2. **统一响应**：标准JSON格式响应
3. **版本控制**：URL中包含版本号（/api/v1/）
4. **安全认证**：JWT Token认证机制

### 6.2 接口分类

#### 6.2.1 前台接口

| 模块 | 接口示例 | 说明 |
|------|----------|------|
| 用户 | POST /api/auth/login | 用户登录 |
| 用户 | POST /api/auth/register | 用户注册 |
| 商品 | GET /api/products | 商品列表 |
| 商品 | GET /api/products/{id} | 商品详情 |
| 购物车 | POST /api/cart | 添加购物车 |
| 订单 | POST /api/orders | 创建订单 |

#### 6.2.2 后台接口

| 模块 | 接口示例 | 说明 |
|------|----------|------|
| 商品 | POST /api/admin/products | 添加商品 |
| 商品 | PUT /api/admin/products/{id} | 修改商品 |
| 订单 | GET /api/admin/orders | 订单列表 |
| 订单 | PUT /api/admin/orders/{id}/ship | 订单发货 |
| 用户 | GET /api/admin/users | 用户列表 |
| 统计 | GET /api/admin/statistics/sales | 销售统计 |

### 6.3 统一响应格式

```json
{
  "code": 200,
  "message": "操作成功",
  "data": { }
}
```
