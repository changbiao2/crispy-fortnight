## 1. 引言

### 1.1 编写目的

本文档是网上购物网站项目的需求规约，旨在明确系统的功能需求、非功能需求，为系统设计和开发提供依据，同时作为项目验收的参考标准。

### 1.2 项目背景

随着电子商务的快速发展，网上购物已成为人们日常生活的重要组成部分。本项目旨在开发一个功能完善、用户体验良好的B2C网上购物网站，满足用户在线购物、商家管理商品和订单的需求。

### 1.3 术语定义

| 术语 | 说明 |
|------|------|
| B2C | Business-to-Consumer，企业对消费者的电子商务模式 |
| 购物车 | 用户临时存放选购商品的虚拟容器 |
| SKU | Stock Keeping Unit，库存量单位，即商品的最小库存单元 |
| JWT | JSON Web Token，用于用户身份认证的令牌 |
| DFD | Data Flow Diagram，数据流图 |

- 《网上购物网站项目计划书》

## 2. 项目概述

### 2.1 项目目标

开发一个功能完善的网上购物网站，包括：
- 前端用户购物系统：商品浏览、搜索、购物车、订单管理
- 后台管理系统：商品管理、订单管理、用户管理、数据统计分析

### 2.2 系统范围

#### 2.2.1 包含功能
- 用户注册、登录、个人信息管理
- 商品展示、搜索、分类浏览
- 购物车管理
- 订单生成、支付、物流跟踪
- 留言系统、在线客服
- 后台商品、订单、用户、库存管理
- 数据统计分析

#### 2.2.2 不包含功能
- 在线支付接口（模拟支付流程）
- 物流系统对接（模拟物流状态）
- 移动端APP（仅支持响应式Web）

### 3.2 角色说明

#### 3.2.1 普通用户（买家）
- **描述**：在网站上浏览商品、购买商品的个人用户
- **权限**：
  - 浏览商品、搜索商品
  - 管理购物车
  - 提交订单、查看订单
  - 管理个人信息、收货地址
  - 发表留言、参与在线客服对话

#### 3.2.2 管理员
- **描述**：负责网站后台管理的操作人员
- **权限**：
  - 商品增删改查、上下架管理
  - 订单查询、发货处理
  - 用户管理（查看、禁用/启用）
  - 库存管理（入库、出库、预警设置）
  - 数据统计分析、报表导出

#### 3.2.3 客服人员
- **描述**：负责与用户在线沟通的客服人员
- **权限**：
  - 回复用户在线消息
  - 查看用户留言并回复
  - 查看用户订单信息（辅助解答）

### 4.2 用户注册登录模块

#### 4.2.1 功能描述
提供用户注册、登录、找回密码等功能。

#### 4.2.2 功能列表

| 功能编号 | 功能名称 | 功能描述 | 优先级 |
|----------|----------|----------|--------|
| UR-001 | 用户注册 | 用户填写用户名、密码、邮箱/手机完成注册 | 高 |
| UR-002 | 邮箱验证 | 注册时发送验证码到邮箱进行验证 | 中 |
| UR-003 | 手机验证 | 注册时发送短信验证码进行验证 | 中 |
| UR-004 | 用户登录 | 使用用户名和密码登录系统 | 高 |
| UR-005 | 记住密码 | 登录时选择记住密码，下次自动填充 | 中 |
| UR-006 | 找回密码 | 通过邮箱或手机验证重置密码 | 高 |
| UR-007 | 退出登录 | 用户主动退出登录状态 | 高 |

#### 4.2.3 业务流程

**用户注册流程**：
1. 用户进入注册页面
2. 填写用户名、密码、确认密码
3. 填写邮箱或手机号
4. 获取并输入验证码
5. 系统验证信息合法性
6. 创建用户账号
7. 注册成功，跳转登录页

**用户登录流程**：
1. 用户进入登录页面
2. 输入用户名和密码
3. 系统验证用户名密码
4. 验证通过，生成JWT Token
5. 登录成功，跳转首页

## 5. 非功能需求

### 5.1 性能需求

| 需求编号 | 需求描述 | 指标 |
|----------|----------|------|
| NF-001 | 页面加载时间 | 首屏加载 ≤ 3秒 |
| NF-002 | 接口响应时间 | 普通接口 ≤ 500ms，复杂查询 ≤ 2s |
| NF-003 | 并发用户数 | 支持 ≥ 1000 并发用户 |
| NF-004 | 系统吞吐量 | 支持 ≥ 100 TPS |

### 5.2 可用性需求

| 需求编号 | 需求描述 | 指标 |
|----------|----------|------|
| NF-005 | 系统可用性 | ≥ 99.9% |
| NF-006 | 故障恢复时间 | ≤ 30分钟 |
| NF-007 | 数据备份 | 每日自动备份 |

### 5.3 安全性需求

| 需求编号 | 需求描述 |
|----------|----------|
| NF-008 | 用户密码采用加密存储（BCrypt） |
| NF-009 | 敏感接口使用JWT认证 |
| NF-010 | 防止SQL注入、XSS攻击 |
| NF-011 | 接口防重放攻击 |

### 5.4 兼容性需求

| 需求编号 | 需求描述 |
|----------|----------|
| NF-012 | 支持Chrome、Firefox、Safari、Edge浏览器 |
| NF-013 | 支持响应式布局，适配PC、平板、手机 |
